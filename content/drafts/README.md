# Drafts

Working article drafts generated by AI assistants.

## Purpose
This folder stores in-progress article files that are not yet review-ready.

## Article Filename Format (Required)
Use this format for all draft files:

`YYYY-MM-DD__topic-slug__vNN.md`

Example:
- `2026-04-30__ai-governance-auditability__v01.md`

Rules:
- `YYYY-MM-DD` is the draft creation date in UTC.
- `topic-slug` is lowercase kebab-case and must be stable across revisions.
- `vNN` is a two-digit revision number (`v01`, `v02`, ...).
- Use `.md` extension only.
- Do not include spaces, uppercase letters, or special characters outside hyphens/underscores used in the format.

## Required Front Matter
Each draft must include YAML front matter with at least:

- `article_id` (stable ID, e.g., `ART-2026-0001`)
- `title`
- `slug`
- `status` (must be one of draft review status values below)
- `created_utc`
- `updated_utc`
- `author`
- `review_owner` (human)
- `canonical_url` (placeholder allowed per canonical rules)
- `seo_title`
- `seo_description`
- `seo_keywords`
- `wp_categories`
- `wp_tags`

## Draft Review Status Values
Allowed `status` values while in this folder:
- `draft-in-progress`
- `draft-ready-for-review`
- `draft-needs-revision`

## Canonical URL Placeholder Rules
- If final canonical URL is not yet known, use:
  - `canonical_url: "TBD_CANONICAL_URL"`
- Do not invent production URLs.
- Replace placeholder during review/approval before WordPress draft push.

## Boundaries
- Drafts may be incomplete and editable.
- Do not treat this folder as publication-ready.
- Move to `content/ready-for-review/` only after status is `draft-ready-for-review` and required front matter is present.
