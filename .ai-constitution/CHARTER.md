# AI Constitution Repository Charter

## Purpose
This repository operationalizes the AI Constitution doctrine as enforceable project governance. The doctrine source of truth is `docs/source-framework/`.

## Constitutional Priorities
1. Structure before scale
2. Authority before autonomy
3. Transparency before trust
4. Audit before deployment confidence
5. Human oversight above automated execution
6. Retrieval is not authority
7. Architecture determines outcomes

## Scope
This charter applies to all code, automation, workflows, documents, and operational decisions in this repository.

## Governance Controls
- **Authority Gate:** Every material change must name an accountable human owner.
- **Transparency Gate:** Significant decisions must include rationale and traceable references.
- **Audit Gate:** Releases require a completed governance checklist and decision log updates.
- **Oversight Gate:** High-impact automations must include explicit human review steps.

## Change Policy
- Changes to constitutional governance files require at least one human reviewer.
- Doctrine interpretation changes must include a decision log entry.
- No deployment should bypass governance artifacts.
